# FE 요구사항 정의서 - Claude Code 입력용

> 목적: 이 문서는 업로드된 FE 요구사항 정의서를 Claude Code가 바로 이해하고 구현 컨텍스트로 사용할 수 있도록 정리한 Markdown 명세서입니다.  
> 원문 TSV/텍스트를 기준으로 중복 기능 ID를 제거했고, 구현에 필요한 화면 흐름, 상태, 라우트, 데이터 모델, API 훅 후보, 수용 기준을 함께 정리했습니다.

## 0. 문서 사용 방법

Claude Code에 이 파일을 넣을 때는 다음 기준으로 작업을 지시합니다.

1. **우선순위는 `MVP` + `높음` 기능부터 구현**한다.
2. `추가기능`은 라우트/컴포넌트 뼈대까지 만들어도 되지만, 실제 상세 구현은 MVP 완료 후 진행한다.
3. 백엔드 API가 아직 없으면 `services/api/*` 또는 `mocks/*`에 mock adapter를 만들고, 컴포넌트는 실제 API 교체가 쉽도록 hook/service 레이어를 사용한다.
4. 화면 텍스트, 버튼명, 에러 문구는 기본적으로 한국어로 작성한다.
5. 원문 요구사항에 명시되지 않은 정책은 `기획 확인 필요`로 표시하고, 임의 확정하지 않는다.

## 1. 프로젝트 개요

이 서비스는 사용자의 게임 성향을 분석하고, Steam 연동 데이터와 설문 데이터를 활용하여 맞춤형 게임을 추천하는 프론트엔드 서비스이다. 주요 기능 축은 다음과 같다.

- 로그인/회원가입 및 JWT 기반 인증
- Steam 계정 연동, Steam 라이브러리 동기화, 비공개/실패 예외 처리
- 성향 설문, 실시간 응답 저장, 6대 성향 결과 시각화
- 메인 페이지, 검색/탐색, 게임 추천, 게임 상세
- 마이페이지/유저페이지/팔로우
- 커뮤니티, 게시글, 댓글, 파티 모집
- 고객센터 챗봇

## 3. 전역 사용자 상태와 분기

### 3.1 사용자 인증 상태

| 상태 | 설명 | 대표 화면/기능 |
|---|---|---|
| `anonymous` | 비로그인 사용자 | 메인 일반 추천, 로그인/회원가입, 검색 일부 |
| `emailSignupPending` | 이메일 회원가입 입력 중 | 회원가입 폼, 이메일 인증, 닉네임 중복확인 |
| `authenticated` | JWT 인증 완료 | 메인 개인화 추천, 설문, 추천, 마이페이지 |
| `steamLinkPending` | Steam 연동 진행 중 | Steam OpenID 인증, 연동 안내/콜백 |
| `steamLinked` | Steam 연동 완료 | 라이브러리 요약, 플레이 기반 추천, Steam 프로필 표시 |
| `steamPrivateOrFailed` | Steam 정보 조회 실패/비공개 | 비공개 안내, 재시도, 설문 기반 추천 모드 |
| `surveyIncomplete` | 설문 미완료/건너뜀 | 메인 배너의 설문 이어하기, 기본 추천 |
| `surveyCompleted` | 설문 완료 | 성향 결과, 추천 게임, 레이더 차트 |

### 3.2 공통 UI 정책

- 모든 비동기 요청에는 `loading`, `success`, `error`, `empty` 상태를 구분한다.
- API 실패 시 사용자가 다음 액션을 선택할 수 있어야 한다. 예: 재시도, 홈으로 이동, 설문 기반 모드로 진행.
- 버튼 클릭으로 페이지 이동이 발생하는 기능은 라우트 이동과 함께 접근 권한을 확인한다.
- 로그인 필요 기능은 비로그인 상태에서 로그인 페이지로 이동시키고, 로그인 후 원래 목적지로 복귀할 수 있도록 `redirect` 값을 보존한다.
- MVP 화면은 최소한의 반응형 레이아웃을 지원한다. 모바일/태블릿/데스크톱에서 카드 리스트, 폼, 차트가 깨지지 않아야 한다.

## 5. 핵심 플로우

### 5.1 이메일 로그인 플로우

1. 사용자가 `/login`에서 이메일과 비밀번호를 입력한다.
2. 클라이언트는 필수 입력값과 이메일 형식을 검증한다.
3. 로그인 API를 호출하고 성공 시 JWT를 저장한다.
4. `자동 로그인` 옵션이 선택되어 있으면 지속 저장소에 인증 유지 정보를 저장한다.
5. 로그인 성공 후 기본 이동지는 홈이며, `redirect`가 있으면 해당 경로로 이동한다.
6. 실패 시 원인 안내 메시지를 출력한다.

### 5.2 이메일 회원가입 플로우

1. 사용자가 이메일, 비밀번호, 비밀번호 확인, 닉네임, 생년월일, 성별을 입력한다.
2. 이메일 인증 코드 발송 및 6자리 인증 코드 검증을 제공한다.
3. 닉네임 중복확인을 완료해야 한다.
4. 비밀번호는 10자 이상이며 영문과 숫자를 포함해야 한다.
5. 만 15세 미만이면 회원가입을 제한하고 에러 문구를 출력한다.
6. 회원가입 완료 후 로그인 또는 온보딩/Steam 연동 안내로 이동한다.

### 5.3 Steam 로그인/연동 플로우

1. Steam 소셜 로그인 사용자는 Steam OpenID/OAuth 인증으로 회원가입 또는 로그인을 진행한다.
2. 최초 가입 사용자는 Steam UID 기반 회원 정보를 생성하고 Steam 연동 및 온보딩 플로우로 이동한다.
3. 이메일 가입/로그인 사용자는 로그인 완료 후 Steam 라이브러리 연동 안내 페이지에서 `연동하기` 또는 `건너뛰기`를 선택한다.
4. 연동 성공 시 Steam 프로필 정보, 가져온 게임 수, 플레이 데이터 기반 성향 분석 결과를 보여준다.
5. 연동 실패 또는 비공개 상태이면 실패 원인, 재시도 버튼, 설문 기반 추천 모드 이동 버튼을 제공한다.
6. 마이페이지 또는 설정에서 Steam 플레이 기반 자동 추천 기능을 활성화/비활성화할 수 있어야 한다.

### 5.4 설문/성향 결과 플로우

1. `/survey/intro`에서 총 문항 수, 예상 소요 시간, 성향 축 정보를 안내한다.
2. 사용자가 `시작하기`를 누르면 설문이 시작된다.
3. 설문 문항은 1~5점 척도이며, 선택 시 다음 문항으로 자동 이동한다.
4. 상단에 현재 문항 번호와 전체 진행 상태를 표시한다. 완료는 주황색, 미진행은 회색이다.
5. 이전 문항 이동, 이전 응답 재선택, 현재 문항 건너뛰기를 제공한다.
6. 응답 결과는 실시간 저장한다.
7. 완료 후 6대 성향 점수, 레이더 차트, 대표 성향 타이틀/설명/태그를 보여준다.
8. 결과 화면에서 `추천 더 보러가기` 또는 `메인으로` 이동할 수 있다.

### 5.5 추천/검색/상세 플로우

1. 메인 페이지는 비로그인 사용자에게 일반 추천/신규 게임을 제공하고, 로그인 사용자에게 성향 기반 추천과 매칭률을 제공한다.
2. 검색 페이지는 게임명 검색, 최근 검색 삭제, 장르/태그/가격/세일/자막/플레이 모드/평점 필터를 제공한다.
3. 검색 결과가 없으면 `해당하는 게임이 없어요` 안내와 대체 추천을 제공한다.
4. 추천 페이지는 이번 주 인기게임, 평균 4점 이상 선호 기반 추천, 선호 태그/장르, 세일 중인 인기 게임, 취향 재분석 이동을 제공한다.
5. 게임 상세는 장르, 플레이 모드, 연령 등급, 한글 자막, 최소/권장 사양, Steam 리뷰/평점, 가격, 구매 링크, 위시리스트, 라이브러리 추가, 비슷한 게임 추천을 제공한다.

## 12. 수용 기준 작성 규칙

Claude Code가 테스트 또는 구현 완료 기준을 만들 때 다음 형식을 사용한다.

```txt
Given: 사용자가 특정 상태에 있다.
When: 사용자가 특정 액션을 한다.
Then: 화면/상태/API 호출/라우팅 결과가 요구사항과 일치한다.
```

### 예시

```txt
Given 비로그인 사용자가 로그인 페이지에 접근했다
When 이메일과 비밀번호를 입력하고 로그인 버튼을 누른다
Then 로그인 API가 호출되고, 성공 시 JWT가 저장되며 홈으로 이동한다
```

```txt
Given Steam 라이브러리가 비공개인 사용자다
When Steam 연동 결과를 확인한다
Then 비공개 안내 화면, 공개 전환 안내, 다시 시도 버튼, 설문 기반 추천 모드 버튼이 노출된다
```

```txt
Given 설문 3번 문항을 보고 있는 사용자다
When 1~5점 중 하나를 선택한다
Then 응답이 저장되고 다음 문항으로 자동 이동한다
```

## 13. 기획 확인 필요 항목

아래 항목은 원문 요구사항만으로 세부 정책이 부족하므로 구현 전 확인이 필요하다.

- Steam 인증 방식 표기: 원문에 `Steam OpenID/OAuth`가 혼용되어 있으므로 실제 백엔드 인증 플로우 확인 필요
- JWT 저장 위치: localStorage/sessionStorage/httpOnly cookie 중 정책 확인 필요
- 자동 로그인 만료 기간 확인 필요
- 이메일 인증 코드 유효시간 구체값 확인 필요
- 성향 축 6개 이름과 점수 계산 방식 확인 필요
- 설문 문항 수: 원문 예시는 7문항이나 실제 문항 수 확인 필요
- 추천 알고리즘 및 매칭률 계산 기준 확인 필요
- Steam 리뷰/평점 데이터 수급 방식 확인 필요
- 게임 가격/세일 정보 데이터 수급 방식 확인 필요
- 마이페이지의 `클리어한 게임`, `중단된 게임` 판정 기준 확인 필요
- 팔로우/맞팔로우 상태 정의 확인 필요
- 커뮤니티 게시글 분류 목록 확인 필요
- 파티 모집의 신청/승인/거절 권한 정책 확인 필요
- 챗봇 API 또는 시나리오 기반 응답 정책 확인 필요
- `POST-FE-004`, `POST-FE-005` 상세 내용 확인 필요
