# PR_GUIDELINES.md — Pull Request 가이드 (v2)

> 이 프로젝트의 작업 모드는 **풀팀 협업** — PR 의무, 본인 머지 금지, 최소 1명 승인.

## 작업 모드별 PR 정책

| 모드 | PR 의무 | 본인 머지 | 푸시 전 승인 | 머지 게이트 |
|---|---|---|---|---|
| 단독 | normal/cross만 | 허용 | 자동 | 본인 |
| 소팀 | 모든 흐름 | 허용(리뷰 권장) | 보고 | 본인 또는 1명 |
| **풀팀 (이 프로젝트)** | **모든 흐름** | **금지** | **승인** | **최소 1명 승인** |

## PR 생성 원칙

- 1 티켓 = 1 PR
- 검증 통과 후 생성
- PR 본문은 자동 작성 (티켓 + 회고 섹션 기반)

## PR 제목 양식

```
<type>(<scope>): <티켓-ID 또는 짧은 설명>
```
예: `feat(auth): LOGIN-FE-001 이메일 로그인 화면 구현`

## PR 본문 (자동 생성)

`.github/PULL_REQUEST_TEMPLATE.md` 양식. 자동 채워지는 항목:
- 관련 티켓 링크 (REQ 기능 ID)
- 변경 요약 (티켓 회고 섹션 기반)
- 변경 파일 목록
- 자동 검증 결과 체크리스트
- 리뷰 포인트
- 스크린샷 (UI 변경 시 — 데스크탑 다크 모드)

## PR 크기 가이드

| 크기 | 변경 | 권장 |
|---|---|---|
| 작음 | ~200줄 | 권장 |
| 중간 | 200~500줄 | OK |
| 큼 | 500줄+ | 분할 고려 |
| 거대 | 1000줄+ | 분할 권장 |

## 리뷰 체크리스트 (PR 본문에 자동 포함)

리뷰어용 (풀팀 필수):
- [ ] 티켓 AC가 모두 만족됐는가
- [ ] 검증 단계 모두 통과
- [ ] 영향 범위가 티켓 Plan과 일치
- [ ] 자동 생성 파일 안 건드림
- [ ] semantic token만 사용 (primitive/hex/인라인 style 없음)
- [ ] 폴더 README 갱신
- [ ] 환경변수 추가 시 .env.local.example 동기화

작성자용:
- [ ] 티켓 `## 회고` 섹션 채움
- [ ] 변경된 폴더 README 업데이트
- [ ] 자동 검증 통과

## 머지 전략

- **Squash and merge** 권장
- 머지 후 브랜치 자동 삭제 권장 (GitHub 설정)
- **풀팀: 본인 머지 금지 — 리뷰어가 승인 후 머지**

## 자동화

`pr-creating` 스킬이 자동: 검증 통과 확인 → 푸시(승인) → PR 본문 작성 → **승인** → `gh pr create` → URL 보고. **머지는 본인이 하지 않음.**

## hotfix 흐름 (예외)

- 출발점: main / PR 대상: main / 머지 후 dev로 cherry-pick / 더 엄격한 리뷰 (2명+ 승인 권장)

## Forbidden

### 모든 모드
- 자동 검증 실패 상태에서 PR 생성 ❌
- 회고 섹션 비운 상태에서 PR 생성 ❌
- PR 본문 비우기 ❌

### 풀팀 추가
- 본인 머지 ❌
- 리뷰 없이 머지 ❌
